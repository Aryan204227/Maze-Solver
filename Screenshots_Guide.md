# 📸 Screenshots Guide for Maze Solver Project

This guide outlines the screenshots to capture for your project report, presentation slides, and documentation, ensuring your project evaluation goes smoothly.

---

## 1. Required Screenshots List

### 🖥️ Screen 1: App Launch & Splash Screen
*   **When to capture:** Immediately upon running the application (`java -cp bin com.mazesolver.Main`).
*   **What to capture:** The borderless `SplashScreen` showing the loading progress bar and animated startup text.
*   **Purpose:** Demonstrates a premium, commercial-grade application launch.

### 🏠 Screen 2: Welcome Dashboard Panel (Dark Theme)
*   **When to capture:** Right after the splash screen finishes loading and opens the home dashboard.
*   **What to capture:** The entire main window highlighting the title, feature cards grid (Generate, Solve, Settings, Themes), and the green-gradient "Open Maze Editor" button.
*   **Purpose:** Highlights modern UI styling, layouts, and typography.

### 🗺️ Screen 3: Welcome Dashboard Panel (Light Theme)
*   **When to capture:** Click **Dark Mode / Light Mode** in the sidebar while on the dashboard.
*   **What to capture:** The Light theme version of the home dashboard.
*   **Purpose:** Shows complete theme styling capabilities.

### ✏️ Screen 4: Maze Editor Mode (Drawing Custom Walls)
*   **When to capture:** Click **Open Maze Editor**, select **Edit** from the control toolbar, and draw some walls by clicking and dragging.
*   **What to capture:** The grid showing custom walls with the yellow "EDIT MODE" text active.
*   **Purpose:** Demonstrates grid coordinate mouse tracking.

### 🌀 Screen 5: Solving Process in Real-Time
*   **When to capture:** Click **Solve** with a medium speed delay.
*   **What to capture:** The grid mid-solve, showing the blue exploring path, purple visited nodes, and orange backtrack cells.
*   **Purpose:** Highlights the visual pathfinding process.

### 🟢 Screen 6: Multiple Solutions & Completed State
*   **When to capture:** Let the solver finish searching the maze.
*   **What to capture:** The final completed maze with the shortest path highlighted in green, the Stats panel displaying metrics (total solutions, visited nodes, max depth), and the status bar reading "Solving completed!".
*   **Purpose:** Confirms path finding, shortest path calculation, and telemetry tracking.

### 💾 Screen 7: File Dialog Interactions
*   **When to capture:** Click **Save** or **Load** from the toolbar.
*   **What to capture:** The native `JFileChooser` dialog showing filtering for `.maze` files.
*   **Purpose:** Highlights persistence integration.

---

## 💡 Quick Tips for Clean Screenshots
- **Maximize the Window:** Ensure the window is maximized to show the grid and sidebar cleanly.
- **Set the Speed Slider:** Set the speed slider to middle/slow values to capture clear real-time screenshots of the solver in action.
- **Use Windows Snipping Tool:** Press `Windows + Shift + S` to capture clean window snippets without background desktop clutter.
